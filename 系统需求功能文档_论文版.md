# SmartLedger 系统需求功能文档（论文版）

> 基于 Spring Boot + Vue 的大学生生活费记账与超前消费预警系统
> 文档版本：v2.0 | 生成日期：2026年4月12日

---

## 一、系统概述

### 1.1 系统名称
**SmartLedger —— 基于 Spring Boot + Vue 的大学生生活费记账与超前消费预警系统**

### 1.2 系统定位
本系统面向大学生群体，围绕"按月领生活费、易超支、难规划"的真实需求，提供账单管理、预算控制、消费预警、数据分析等核心功能，帮助大学生建立理性消费习惯，避免超前消费。

### 1.3 核心创新点
1. **消费前预警机制**：记账前系统根据当前余额、剩余天数、消费类型给出红/黄/绿灯建议，防患于未然
2. **三级消费属性分类**：将消费划分为"生存必需/改善生活/欲望消费"三级，引导理性消费
3. **校园场景化设计**：食堂套餐快捷记账、AA 拼单结算、存钱计划等贴近大学生活的功能

## 五、核心算法说明

### 5.1 消费前预警算法
```
输入：消费金额 amount, 分类属性 attribute, 当前已消费 currentSpent, 月度预算 monthlyBudget, 剩余天数 remainingDays

步骤：
1. 计算剩余额度：remaining = monthlyBudget - currentSpent
2. 计算每日可用：dailyBudget = remaining / remainingDays
3. 判断逻辑：
   - 如果 attribute == 1（生存必需）: 绿灯，建议消费
   - 如果 amount > remaining: 红灯，强烈不建议（将超支）
   - 如果 budgetUsage >= 80% 且 attribute >= 2: 黄灯，预警期非必需
   - 如果 attribute == 3（欲望）且 amount > dailyBudget * 2: 黄灯，大额欲望消费
   - 其他情况：绿灯，可以消费
4. 超支风险预测：
   - 日均消费速度 = currentSpent / (30 - remainingDays)
   - 预计月末消费 = 日均消费速度 * 30
   - 预计超支 = max(0, 预计月末消费 - monthlyBudget)
```

### 5.2 必需/非必需判定规则
- 从 `categories.default_attribute` 字段获取
- 1 = 生存必需（食堂、水电、交通、医疗等）
- 2 = 改善生活（水果、服饰、图书、运动等）
- 3 = 欲望消费（奶茶、游戏、盲盒、追星等）

### 5.3 消费评级算法（待实现）
```
输入：月度消费数据

评分维度：
1. 必需消费占比（权重 30%）：60%-80% 为最佳
2. 欲望消费占比（权重 25%）：< 20% 为优秀
3. 预算使用率（权重 25%）：70%-90% 为合理
4. 消费频率稳定性（权重 20%）：标准差越小越好

评级标准：
- S 级（90-100 分）：极简消费主义
- A 级（75-89 分）：消费达人
- B 级（60-74 分）：理性消费者
- C 级（40-59 分）：需要控制消费
- D 级（< 40 分）：消费无度
```

## 十、系统特色总结

1. **贴近大学生活场景**：食堂套餐快捷记账、校园卡消费等
2. **超前消费预警**：消费前三色灯机制，防患于未然
3. **三级消费分类**：生存必需 / 改善生活 / 欲望消费，引导理性消费
4. **存钱计划功能**：帮助大学生养成储蓄习惯
5. **趣味排行系统**：消费评级 + 成就系统，增强用户粘性
6. **完整的管理后台**：用户管理、账单管理、数据统计
7. **响应式设计**：适配桌面端和移动端
8. **一屏展示设计**：核心页面（预警、排行）一屏完成，无需滚动
